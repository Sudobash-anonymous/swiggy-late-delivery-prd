# Wireframes: Swiggy Late Delivery Features

## Overview
4 low-fidelity wireframes showing the key user flows for late delivery reduction.

---

## Wireframe 1: Customer Order Tracking
**File:** `wireframe-1-customer-tracking.png`

**What it shows:**
- Before vs After comparison
- Proactive delay notification
- ₹25 coupon auto-added
- Cancel option (no fee)

**Key elements:**
- Yellow warning banner
- Old ETA vs New ETA
- Confidence indicator (High/Medium/Low)

---

## Wireframe 2: Rider App — Delay Triage
**File:** `wireframe-2-rider-app.png`

**What it shows:**
- Rider options when restaurant is delayed
- "No penalty" reassurance
- Reassign button

**Key elements:**
- Restaurant delay message
- 4 options (Wait / Reassign / Mark delay / Traffic)
- Penalty notice

---

## Wireframe 3: Support Dashboard (Internal)
**File:** `wireframe-3-support-dashboard.png`

**What it shows:**
- Real-time metrics (late orders, rate, wait time)
- Root cause breakdown (bar chart)
- Recent late orders table

**Key elements:**
- 4 KPI cards
- Root cause visualization
- Editable table with actions

---

## Wireframe 4: Restaurant Prep Score
**File:** `wireframe-4-restaurant-score.png`

**What it shows:**
- Score out of 100 (78 shown)
- Commission impact (+1% penalty)
- Top delay causes

**Key elements:**
- Gauge visualization
- Commission impact message
- Improvement tips link
